- **Camera**: Smartphone (iPhone/Android) is perfect

## Filming Schedule (2-3 Hours Total)

### Hour 1: Setup & Test Shots
- Set up lighting and camera
- Test audio levels
- Record test clips
- Review and adjust

### Hour 2: Record All Scenes
- Film yourself on camera (all scenes)
- Do 3-5 takes of each scene
- Record screen captures
- Capture B-roll (you coding, thinking, etc.)

### Hour 3: Editing
- Import all footage
- Cut together following script
- Add voiceover where needed
- Add music and transitions
- Export and upload

## Screen Recording Tips

### What to Record:
1. **Dashboard Tour** (30 sec)
   - Smooth mouse movements
   - Click through features
   - Show polished UI

2. **Code Walkthrough** (20 sec)
   - Open VS Code
   - Show monorepo structure
   - Scroll through key files
   - Show terminal commands

3. **Live Features** (30 sec)
   - WebSocket connection
   - Real-time updates
   - Smooth animations
   - Interactive elements

4. **Architecture** (10 sec)
   - Diagram of system
   - Tech stack logos
   - Database schema

### Recording Settings:
- **Resolution**: 1920x1080 (1080p)
- **Frame Rate**: 30fps or 60fps
- **Format**: MP4
- **Tools**: OBS Studio (free), QuickTime (Mac), Xbox Game Bar (Windows)

---

## Audio Recording Tips

### Voiceover Recording:
1. **Environment**: Quiet room, close doors, turn off fans
2. **Microphone**: 6-8 inches from mouth
3. **Delivery**: Clear, confident, passionate (not monotone)
4. **Pacing**: Speak slightly slower than normal
5. **Energy**: High energy, but not shouting

### Script Reading Tips:
- Read script 5 times before recording
- Record in small chunks (30 sec each)
- Do 3 takes of each chunk
- Take breaks to rest your voice
- Smile while talking (it changes your tone)

## Editing Workflow

### Software Options:
- **Free**: DaVinci Resolve, iMovie, CapCut
- **Paid**: Adobe Premiere, Final Cut Pro

### Editing Steps:

1. **Import All Footage** (10 min)
   - Organize by scene
   - Label clips clearly

2. **Rough Cut** (30 min)
   - Lay out all clips in order
   - Cut out mistakes and pauses
   - Match to script timing

3. **Add Voiceover** (20 min)
   - Record voiceover for screen recordings
   - Sync with visuals
   - Adjust audio levels

4. **Add Music** (10 min)
   - Background music (low volume, 20-30%)
   - Epic music for opening/closing
   - Fade in/out smoothly

5. **Add Text & Graphics** (15 min)
   - Title cards
   - Your name and project name
   - Links at the end
   - Captions (optional but recommended)

6. **Color Correction** (10 min)
   - Adjust brightness/contrast
   - Make colors pop
   - Consistent look across scenes

7. **Final Review** (10 min)
   - Watch full video 2-3 times
   - Check audio levels
   - Fix any issues

8. **Export** (15 min)
   - Format: MP4
   - Resolution: 1080p
   - Frame Rate: 30fps
   - Quality: High (not maximum, file size matters)

## Pro Tips for Maximum Impact

### Visual:
1. **Show, don't tell** - Let the UI speak for itself
2. **Fast cuts** - Keep energy high, 3-5 seconds per shot
3. **Smooth transitions** - Use fades, not hard cuts
4. **Color grade** - Make it look professional
5. **Captions** - Many people watch without sound

### Audio:
1. **Clear voice** - Enunciate, don't mumble
2. **Confident tone** - You believe in this, show it
3. **Varied pacing** - Speed up for excitement, slow down for emphasis
4. **Music volume** - Background only, 20-30% of voice
5. **No dead air** - Always something happening

### Content:
1. **Hook in 5 seconds** - Grab attention immediately
2. **Show working features** - Prove it's real
3. **Show code** - Prove you're technical
4. **Show passion** - Prove you care
5. **End with CTA** - Tell them what to do next

### Authenticity:
1. **Be yourself** - Don't try to sound corporate
2. **Show the grind** - Late night coding shots
3. **Admit it's unfinished** - Honesty is powerful
4. **Show confidence** - You know this will work
5. **Show vision** - Paint the future
